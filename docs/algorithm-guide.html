ภาพรวมระบบ
System Overview

CargoFit คำนวณการจัดวางกล่องสินค้าลงในตู้คอนเทนเนอร์โดยอัตโนมัติ โดยพยายามให้ได้จำนวนกล่องมากที่สุดภายใต้ข้อจำกัดด้านขนาดและความสูงที่รับน้ำหนักได้ ผลลัพธ์แสดงเป็นภาพ 3 มิติเพื่อให้ทีมขนส่งวางแผนล่วงหน้า

🔢 ข้อมูลที่รับ

  • ประเภทตู้คอนเทนเนอร์
  • รายการสินค้า + ขนาด + Pattern
  • จำนวนกล่องที่ต้องการบรรจุ

📐 ผลลัพธ์ที่ส่งออก

  • ตำแหน่ง (X, Y, Z) ของกล่องทุกกล่อง
  • สถิติจำนวนที่บรรจุได้จริง
  • ภาพ Isometric 3 มิติ

Glossary
คำศัพท์ความหมาย
Stack (กอง)กลุ่มกล่องซ้อนกันในแนวตั้ง ณ ตำแหน่ง Y เดียวกัน แต่ละกองมีหมายเลข StackIndex เริ่มจาก 0
Layer (ชั้น)ระดับความสูงหนึ่งๆ ของกอง — ชั้น 0 = พื้น, ชั้น 1 = เหนือขึ้นไป มี LayerIndex
Pattern A / Bรูปแบบการจัดเรียงกล่องต่อชั้น — ชั้นเลขคู่ใช้ A, ชั้นเลขคี่ใช้ B เพื่อสลับทิศทางเพิ่มความเสถียร
Sectionกลุ่มย่อยภายใน Pattern หนึ่งชั้น วางต่อกันในแนวกว้าง (X)
SubRowแถวย่อยภายใน Section ที่มีการหมุนกล่อง (Rotated) ต่างกัน
Primary Zoneพื้นที่หลักจาก Y=0 ถึง L−50 cm สำหรับจัดกองสินค้าหลัก
Condo Zoneพื้นที่หน้าตู้ ~50 cm สำหรับวางกล่องส่วนเกินจาก Primary เป็นคอลัมน์ตรงๆ
Scatterการนำกล่องที่เหลือหลัง Condo วางเสริมทับยอดกองเดิม (เฉพาะสินค้าชนิดเดียวกัน)
MaxLayersชั้นสูงสุดต่อกองที่อนุญาต กำหนดตามน้ำหนักสินค้าและความปลอดภัย
EffectiveLayersชั้นที่ระบบคำนวณให้ใช้จริง เพื่อกระจายหลายสินค้าให้ใช้ความยาวตู้สม่ำเสมอ (≤ MaxLayers)
CBMปริมาตรกล่องในหน่วยลูกบาศก์เมตร = W × L × H ÷ 1,000,000
Tier Replicationการทำซ้ำชุด Pattern ในแนวกว้าง X เมื่อตู้กว้างพอจะใส่ได้มากกว่า 1 ชุด

6-Phase Packing Algorithm

Algorithm ทำงานตามลำดับ 8 ขั้น (ขั้น 0–7) โดยแต่ละขั้นรับ Output จากขั้นก่อนหน้า:

PHASE 0 คำนวณจำนวนชั้นที่เหมาะสม ComputeEffectiveLayers

เมื่อมีสินค้าหลายชนิด ถ้าปล่อยให้แต่ละชนิดวางสูงเต็ม MaxLayers พื้นที่ความยาวตู้อาจถูกใช้ไม่หมดหรืออาจล้น ขั้นนี้หา EffectiveLayers ให้รวมพื้นที่ที่ใช้ทั้งหมดอยู่ภายใน targetY = InteriorL − 50 cm

สูตรคำนวณเบื้องต้น

สำหรับสินค้าแต่ละชนิด i:
  bpl[i]        = กล่องต่อชั้น (จาก Pattern A)
  stackDepth[i] = ความลึกของกอง 1 กอง (cm)
  stacks[i]     = ⌈ qty / (bpl × MaxLayers) ⌉

requiredY = Σ stacks[i] × stackDepth[i]
targetY   = InteriorL − 50 cm

ถ้า requiredY ≤ targetY  →  ใช้ MaxLayers ปกติ (ไม่ต้องปรับ)
ถ้า requiredY > targetY  →  ลด EffectiveLayers ด้วย Snap Point Search

Snap Point Search

ระบบสร้างชุด snap points คือความสูงที่เป็นจำนวนเต็มเท่าของความสูงกล่อง แล้วทดลองจากน้อยไปมาก:

สินค้า A สูง 26.7 cm → {26.7, 53.4, 80.1, 106.8, ...}
สินค้า B สูง 15.7 cm → {15.7, 31.4, 47.1, 62.8, ...}

รวมทั้งหมด เรียงน้อย→มาก: {15.7, 26.7, 31.4, 47.1, 53.4, ...}

ทดลองทุก snap Tₕ:
  effectiveLayers[i] = ⌊ Tₕ / H[i] ⌋
  requiredY = Σ ⌈ totalLayers[i] / effectiveLayers[i] ⌉ × stackDepth[i]
  → เลือก Tₕ แรกที่ requiredY ≤ targetY
💡
ทำไมต้อง Snap Points? การจำกัดด้วยจำนวนเต็มชั้นทำให้กองสินค้าต่างชนิดมีความสูงใกล้เคียงกัน หากใช้การ scale แบบทั่วไปจะได้ชั้นครึ่งซึ่งในความเป็นจริงเป็นไปไม่ได้
PHASE 1 การจัดวางหลัก Primary Packing

วางกองสินค้าแต่ละชนิดเรียงจาก ด้านในสุด (Y=0) ออกมาหาประตู สินค้าชนิดแรกในรายการวางก่อน ตามด้วยชนิดถัดไป

ขั้นตอนการวางกอง

  1. คำนวณ stackDepth = ความลึกกอง 1 กอง (ผลรวมความยาวกล่องในแนว Y ตาม Pattern)
  2. วางกองที่ currentY — ซ้อนชั้นตาม Pattern จนครบ EffectiveLayers หรือเต็มความสูงตู้
  3. เลื่อน currentY += stackDepth แล้ววางกองถัดไป
  4. วนซ้ำจนได้กล่องครบหรือ currentY ≥ InteriorL

กฎการสลับ Pattern A/B

🔄
กองเลขคู่ (SI 0, 2, 4, ...) — ชั้น 0=A, 1=B, 2=A, ...
กองเลขคี่ (SI 1, 3, 5, ...) — ชั้น 0=B, 1=A, 2=B, ...
การสลับทิศทางทุกชั้นและทุกกองทำให้โครงสร้างล็อกกันมั่นคงขึ้นระหว่างขนส่ง

Tier Replication

ถ้าความกว้างตู้เกินกว่า 1 ชุด Pattern ระบบทำซ้ำ Pattern ในแนว X อัตโนมัติ:

tierWidth = ผลรวมความกว้างของทุก Section ใน Pattern
numTiers  = ⌊ InteriorW / tierWidth ⌋

ตัวอย่าง Aloe 365:
  Section1 = 6 × 21.9 = 131.4 cm  (ไม่หมุน)
  Section2 = 3 × 33.4 = 100.2 cm  (หมุน)
  tierWidth = 231.6 cm,  InteriorW = 239 cm  →  numTiers = 1
PHASE 2 การปรับสมดุล Balancing (A + B + C)

หลัง Primary อาจยังมีพื้นที่ว่างและกล่องที่ยังไม่ได้วาง ขั้นนี้แบ่งเป็น 3 ส่วน

ส่วน A — เพิ่มกองเติมพื้นที่ว่าง

ขณะที่ currentY < targetY ระบบวนเพิ่มกองใหม่ให้สินค้าที่มีกล่องเหลือ เรียงลำดับจาก CBM มากที่สุดก่อน (สินค้าใหญ่กว่ามีสิทธิ์เติมพื้นที่ก่อน)

ขณะที่ currentY < targetY และมีสินค้าเหลือ:
  สำหรับสินค้าเรียงตาม CBM (มาก→น้อย):
    ถ้ามีกล่องเหลือและตำแหน่งไม่ถูก clip:
      → เพิ่มกองใหม่ที่ currentY
      → currentY = max(currentY, r.EndY)

ส่วน B — ปรับจำนวนชั้นให้ตรง EffectiveLayers

กองสูงเกิน EffectiveLayers

ลบชั้นบนสุดออกจนถึงขีดจำกัด

กองต่ำกว่า EffectiveLayers

เติมชั้นขึ้นไป ถ้ากล่องเพียงพอต่อ 1 ชั้นเต็ม

ส่วน C — อัปเดตสถิติ

นับจำนวนกล่องที่วางได้จริงในพื้นที่ Primary และบันทึกไว้เพื่อขั้นตอนถัดไป

PHASE 3 ปรับสมดุลชั้น Layer Balancing

หากกองของสินค้าชนิดเดียวกันยังมีความสูงต่างกันเกิน 1 ชั้น จะโยกชั้นบนสุดจากกองสูงที่สุดไปเพิ่มให้กองต่ำที่สุด ทำซ้ำจนความต่าง ≤ 1 ชั้น

A:4 ชั้น, B:2 ชั้น → ต่าง 2 (✗)   A:3, B:3 → ต่าง 0 (✓)
PHASE 4 ตัดกองส่วนเกิน Partial Removal

ถ้าโซน Condo แน่น (พื้นที่ <50 cm) ตรวจกองสุดท้ายของแต่ละสินค้า — ถ้ากองนั้นสั้นกว่ากองก่อนหน้าเกิน 1 ชั้น จะถูกลบออก เพื่อให้กล่องไปอยู่ใน Condo/Scatter แทนซึ่งใช้พื้นที่คุ้มกว่า

PHASE 5 เรียงกองตามความสูง Sort Stacks by Height

จัดให้กองที่ สั้นกว่า อยู่ใกล้ประตู (Y มากกว่า) และกองที่ สูงกว่า อยู่ด้านใน (Y น้อยกว่า) ช่วยให้พนักงานมองเห็นสินค้าชั้นล่างและเข้าถึงง่ายกว่า นอกจากนี้ยังทำให้โซน Condo มีความสูงที่ถูกต้องในขั้นถัดไป

PHASE 6 การจัดวางโซนหน้าตู้ Condo Placement

วางกล่องที่ยังเหลือหลัง Primary ลงในพื้นที่หน้าตู้ต่อจาก Primary Zone โดยวางเรียงเป็น คอลัมน์ตรงๆ ไม่มีการสลับ Pattern

เงื่อนไขการเข้าโซน Condo

  1. สินค้านั้นยังมีกล่องค้างเหลือหลัง Primary
  2. จำนวนที่เหลือ ≥ cols × targetLayers (ต้องเต็ม 1 คอลัมน์พอดี)
  3. มีพื้นที่ใน Y เพียงพอ (condoY + colDepth ≤ InteriorL)
⚠️
กล่องที่เหลือ ไม่พอ 1 คอลัมน์เต็ม จะไม่ถูกวางใน Condo แต่จะส่งต่อไป Scatter แทน
targetCondoZ  = ⌊ InteriorH / H_adjacent ⌋ × H_adjacent
targetLayers  = min( ⌊ targetCondoZ / H_product ⌋, ⌊ InteriorH / H_product ⌋ )
condoCap      = cols × targetLayers  (กล่องต่อ 1 คอลัมน์)
PHASE 7 กระจายกล่องที่เหลือทับกองเดิม Scatter Top Placement

ถ้ายังมีกล่องเหลือหลัง Primary + Condo จะนำไปวางทับยอดกองที่มีอยู่แล้ว เฉพาะกล่องสินค้าเดิมทับกองสินค้านั้น (ห้ามข้ามสินค้า)

อัลกอริทึมเลือกกอง

สร้างรายการ "กองที่รับได้" = กองที่ยังไม่เต็มเพดาน InteriorH

วนซ้ำจนกล่องหมดหรือทุกกองเต็ม:
  1. เลือกกองที่มี TopZ ต่ำสุด (สั้นที่สุด)
     ✦ TopZ เท่ากัน → เลือก StackIndex น้อยกว่า
  2. วางชั้นใหม่ที่ยอดกองนั้น (Pattern ตาม A/B ของกองนั้น)
  3. อัปเดต TopZ ของกองนั้น
  4. ถ้ากองเต็ม → นำออกจากรายการ

05
รูปแบบการวางชั้น (Layer Pattern)
Layer Pattern Structure

แต่ละชั้นถูกกำหนดด้วย Pattern ซึ่งประกอบด้วย Section หลายส่วนวางต่อกันในแนวกว้าง (X) แต่ละ Section มี SubRow ที่กำหนดแถว คอลัมน์ และทิศทางการวาง

โครงสร้าง JSON

{
  "PatternA": [
    {
      "SubRows": [
        { "Rows": 2, "Cols": 6, "Rotated": false },  // 2 แถว × 6 คอลัมน์ ไม่หมุน
      ]
    },
    {
      "SubRows": [
        { "Rows": 3, "Cols": 3, "Rotated": true  }   // 3 แถว × 3 คอลัมน์ หมุน 90°
      ]
    }
  ]
}
Fieldความหมาย
Rowsจำนวนแถวในแนว Y (ความลึก)
Colsจำนวนคอลัมน์ในแนว X (ความกว้าง)
Rotated = falseวางปกติ — ด้าน W ในแนว X, ด้าน L ในแนว Y
Rotated = trueหมุน 90° — ด้าน L ในแนว X, ด้าน W ในแนว Y

Real-world Example

Mogu 1000 ML × 540 + Mogu 320 ML × 850 ในตู้สั้น 20 ft

สินค้าขนาด W×L×H (cm)MaxLayersต้องการ
Mogu 1000 ML26.2 × 34.8 × 26.78540 กล่อง
Mogu 320 ML25.8 × 38.5 × 15.713850 กล่อง

ขั้น 0 — ComputeEffectiveLayers

Mogu 1000: bpl≈26, stacks=⌈540/(26×8)⌉=3, depth=3×34.8=104.4 cm
Mogu 320:  bpl≈26, stacks=⌈850/(26×13)⌉=3, depth=3×38.5=115.5 cm

requiredY ≈ 220 cm  <  targetY = 595−50 = 545 cm  ✅ ไม่ต้องลด EffectiveLayers

ขั้น 1 — Primary Packing

Mogu 1000: วาง 3 กอง × 8 ชั้น → บรรจุครบ 540
Mogu 320:  วาง 3 กอง × 13 ชั้น → บรรจุครบ 850

ขั้น 2–4 — Balancing

ปรับให้กองแต่ละกองของสินค้าเดียวกันมีจำนวนชั้นใกล้เคียงกัน ±1 ชั้น

ขั้น 5 — Sort

กองที่มีชั้นน้อยกว่า → เลื่อนไปใกล้ประตูมากขึ้น

ขั้น 6–7 — Condo + Scatter

ถ้ายังมีกล่องเหลือหลัง Primary:
  Condo  → วางคอลัมน์ที่ condoAreaStart จนหมดหรือไม่พอ 1 คอลัมน์เต็ม
  Scatter → เอากล่องที่เหลือทับยอดกองเดิม เลือกกองสั้นสุดก่อน
✅
ผลลัพธ์ที่คาดหวัง: Mogu 1000 ML ครบ 540, Mogu 320 ML ครบ 850 ทุกกล่องอยู่ภายในขอบตู้ ไม่มีกล่องล้น ไม่มีการข้ามสินค้า
