## 1. ภาพรวมระบบ

CargoFit คำนวณว่า **จะวางกล่องสินค้าลงในตู้คอนเทนเนอร์อย่างไรให้ได้จำนวนมากที่สุด** โดยคำนึงถึง:

- ขนาดของกล่องแต่ละสินค้า (กว้าง × ยาว × สูง)
- รูปแบบการจัดวางที่ผู้ใช้กำหนดไว้ต่อชั้น (Pattern A / Pattern B)
- จำนวนชั้นสูงสุดที่รับน้ำหนักได้ (MaxLayers)
- ขนาดภายในของตู้คอนเทนเนอร์

ผลลัพธ์คือรายการตำแหน่ง (X, Y, Z) ของกล่องแต่ละกล่อง ซึ่งนำไปแสดงภาพ 3 มิติในโปรแกรม

```
ตู้คอนเทนเนอร์ (มองจากด้านบน)
                   ← ความกว้าง (X) →
    ─────────────────────────────────
  ↑ │  กอง 0  │  กอง 1  │  กอง 2  │  ← หลัง (ในสุด)
  │ │─────────│─────────│─────────│
ค │ │  กอง 0  │  กอง 1  │  กอง 2  │
ว │ │─────────│─────────│─────────│
า │ │  กอง 0  │  กอง 1  │  กอง 2  │
ม │ │                               │
ย │ │  [โซน Condo / ว่าง]           │
า │ │                               │
ว │ │                               │  ← ประตูหน้า (ด้านเปิด)
  ↓ ─────────────────────────────────
```

## 2. นิยามศัพท์สำคัญ

| คำศัพท์ | ความหมาย |
|---------|----------|
| **Stack (กอง)** | กลุ่มกล่องที่ซ้อนกันในแนวตั้ง ณ ตำแหน่ง Y เดียวกัน แต่ละกองมีหมายเลข `StackIndex` |
| **Layer (ชั้น)** | ระดับความสูงหนึ่งๆ ของกอง เช่น ชั้น 0 = พื้น, ชั้น 1 = เหนือชั้น 0 มี `LayerIndex` |
| **Pattern A / B** | รูปแบบการจัดเรียงกล่องในแต่ละชั้น — ชั้นเลขคู่ใช้ Pattern A, ชั้นเลขคี่ใช้ Pattern B (เพื่อสลับทิศแต่ละชั้น) |
| **Section** | กลุ่มย่อยภายใน Pattern หนึ่งชั้น เช่น "แถว 2 แถว × 6 คอลัมน์" วางต่อกันในแนวกว้าง |
| **Primary Zone** | พื้นที่หลักของตู้ที่จัดกองสินค้า (ด้านในสุดของตู้ไปยังระยะ L−50 cm) |
| **Condo Zone** | พื้นที่ส่วนหน้าตู้ประมาณ 50 cm สำหรับวางกล่องที่เหลือแบบตรงๆ (ไม่สลับ Pattern) |
| **Scatter** | การนำกล่องที่เหลือวางเสริมทับบนยอดของกองเดิม (เฉพาะสินค้าชนิดเดียวกัน) |
| **CBM** | ปริมาตรกล่อง (กว้าง × ยาว × สูง ÷ 1,000,000) หน่วยเป็นลูกบาศก์เมตร |
| **MaxLayers** | จำนวนชั้นสูงสุดที่อนุญาตต่อกอง (จำกัดโดยน้ำหนักสินค้า) |
| **EffectiveLayers** | จำนวนชั้นที่ระบบคำนวณว่าควรใช้จริง เพื่อกระจายสินค้าหลายชนิดให้ใช้ความยาวตู้อย่างสม่ำเสมอ |

### ขั้นที่ 0: คำนวณจำนวนชั้นที่เหมาะสม

**จุดประสงค์:** เมื่อมีสินค้าหลายชนิดในตู้เดียวกัน ถ้าปล่อยให้แต่ละสินค้าวางสูงเต็มที่ (MaxLayers) ความยาวตู้อาจถูกใช้ไม่หมด หรือล้นออก  
ขั้นนี้จะหา **จำนวนชั้นที่เหมาะสม** ให้รวมพื้นที่ที่ใช้ทั้งหมด ≤ ความยาวตู้ − 50 cm (เผื่อโซน Condo)

**วิธีคิด:**

```
สำหรับสินค้าแต่ละชนิด:
  bpl      = กล่องต่อชั้น (Boxes Per Layer) ตาม Pattern A
  stackDepth = ความลึกของกอง 1 กอง (= ความยาวกล่องสะสมตาม Pattern)
  stacks   = ⌈ จำนวนที่ต้องการ / (bpl × MaxLayers) ⌉

requiredY = Σ(stacks × stackDepth) สำหรับทุกสินค้า
targetY   = ความยาวตู้ − 50 cm
```

ถ้า `requiredY ≤ targetY` → ไม่ต้องปรับ, ใช้ MaxLayers ปกติ

ถ้า `requiredY > targetY` → ต้องลด EffectiveLayers ลง

**อัลกอริทึมหา EffectiveLayers (กรณีหลายสินค้า ≥ 2 ชนิด):**

ระบบสร้าง "snap points" คือความสูงที่เป็นจำนวนเต็มเท่าของความสูงกล่อง  
เช่น กล่องสูง 26.7 cm → snap = {26.7, 53.4, 80.1, 106.8, ...}

ทดลองทุก snap point Tₕ จากน้อยไปมาก: หา Tₕ แรกที่  
`Σ ( ⌈totalLayers[i] / ⌊Tₕ/H[i]⌋ ⌉ × stackDepth[i] ) ≤ targetY`

ผลลัพธ์ = จำนวนชั้น = ⌊Tₕ / H[i]⌋ สำหรับสินค้าแต่ละชนิด

> **ตัวอย่าง:** มี 2 สินค้า — A สูง 26.7 cm (MaxLayers 8) และ B สูง 15.7 cm (MaxLayers 13)  
> ระบบอาจคำนวณว่าถ้าจำกัดที่ `Tₕ = 80.1 cm` → A ได้ 3 ชั้น, B ได้ 5 ชั้น  
> รวมความยาวที่ใช้อยู่ในขีดจำกัดพอดี

### ขั้นที่ 1: การจัดวางหลัก (Primary Packing)

**จุดประสงค์:** วางกองสินค้าแต่ละชนิดเรียงจากด้านในสุดของตู้ออกมาหาประตู

**วิธีการ:**

```
currentY = 0  (เริ่มจากผนังหลัง)

สำหรับสินค้าแต่ละชนิด:
  1. คำนวณ stackDepth = ความลึกของกอง 1 กอง
  2. วางกองที่ Y = currentY
  3. ซ้อนชั้นตาม Pattern จนครบ EffectiveLayers หรือเต็มความสูง
  4. ขยับ currentY += stackDepth
  5. วนซ้ำจนได้กล่องครบจำนวนที่ต้องการ หรือหมดพื้นที่

  *** สำคัญ: กองคู่ (0,2,4,...) ใช้ Pattern A ก่อน, กองคี่ (1,3,5,...) ใช้ Pattern B ก่อน
      และสลับ A/B ในแต่ละชั้น — ทำให้แต่ละชั้นหมุนสลับทิศ เพิ่มความเสถียร
```

**แผนผัง (มองจากด้านข้าง — แกน Y คือความยาวตู้):**

```
Y=0 (ผนังหลัง)          Y=595 (ประตู)
│                               │
│ ██████ ██████ ██████          │
│ ██████ ██████ ██████          │  ← ชั้น 2
│ ██████ ██████ ██████          │  ← ชั้น 1
│ ██████ ██████ ██████          │  ← ชั้น 0
│ [กอง0] [กอง1] [กอง2]  [ว่าง] │
│                               │
│◄────── primary zone ─────────►│
```

### ขั้นที่ 2: การปรับสมดุล (Balancing)

ขั้นนี้แบ่งเป็น 3 ส่วนย่อย:

#### ส่วน A: เพิ่มกองเติมพื้นที่ว่าง

หลังจากวางสินค้าทุกชนิดแล้ว ถ้าความยาวตู้ยังมีพื้นที่เหลือ (ก่อนถึง `targetY = L − 50 cm`)  
ระบบจะวนเพิ่มกองใหม่ให้กับสินค้าที่ยังมีจำนวนค้างอยู่ โดยเรียงลำดับจาก **CBM มากที่สุดก่อน**

```
ขณะที่ยังมีพื้นที่ว่าง:
  สำหรับสินค้าที่มีเหลือ (เรียงตาม CBM):
    ถ้าตำแหน่ง currentY ไม่ถูก clip → เพิ่มกองใหม่ที่ currentY
    อัปเดต currentY
```

#### ส่วน B: ปรับจำนวนชั้นให้สม่ำเสมอ

เพื่อให้กองของสินค้าชนิดเดียวกันมีความสูงใกล้เคียงกัน (ห่างกันไม่เกิน 1 ชั้น):

- ถ้ากองไหน **สูงเกิน** EffectiveLayers → ตัดชั้นบนออก
- ถ้ากองไหน **ต่ำกว่า** EffectiveLayers → เติมชั้นขึ้นไป (ถ้าจำนวนสินค้าเพียงพอ)

#### ส่วน C: อัปเดตสถิติ

นับจำนวนกล่องที่วางได้จริงในพื้นที่ Primary และบันทึกไว้

---

### ขั้นที่ 3: ปรับสมดุลชั้น (Layer Balancing)

**จุดประสงค์:** ถ้ายังมีกองที่สูงกว่ากองอื่นเกิน 1 ชั้น ให้โยกย้ายชั้นบนสุดจากกองสูงไปใส่กองต่ำ

```
วนซ้ำจนไม่มีการเปลี่ยนแปลง:
  หากองสูงสุด (maxH) และกองต่ำสุด (minH)
  ถ้า maxH − minH > 1:
    ลบชั้นบนสุดของกองสูง
    เพิ่มชั้นให้กองต่ำ (ถ้า capacity ตรงกัน)
```

**ตัวอย่าง (ก่อน → หลัง):**

```
ก่อน:           หลัง:
████  ←ชั้น 3   ████  ████
████            ████  ████
████  ████      ████  ████
████  ████      ████  ████
[A]   [B]       [A]   [B]

กอง A สูง 4 ชั้น, กอง B สูง 2 ชั้น → โยกชั้น 3 ของ A ไปเป็นชั้น 2 ของ B
```

---

### ขั้นที่ 4: ตัดกองส่วนเกิน (Partial Removal)

**จุดประสงค์:** ถ้าโซน Condo มีพื้นที่น้อยกว่า 50 cm (พื้นที่แน่น) ให้ตรวจสอบว่ากองสุดท้าย  
ของแต่ละสินค้ามีชั้นน้อยกว่ากองก่อนหน้ามากกว่า 1 ชั้นหรือไม่ ถ้าใช่ → ลบกองนั้นออก

**เหตุผล:** กองที่มีชั้นน้อยมากอาจทำให้ทรงพังหรือไม่คุ้มกับพื้นที่ที่เสียไป ดีกว่านำกล่องไปรวมกับโซน Condo หรือ Scatter แทน

---

### ขั้นที่ 5: เรียงกองตามความสูง (Sort Stacks by Height)

**จุดประสงค์:** จัดให้กองที่ **สั้นกว่า** อยู่ใกล้ประตู (Y สูง) และกองที่ **สูงกว่า** อยู่ด้านใน (Y ต่ำ)

```
ก่อน:                    หลัง:
ผนังหลัง → ประตู         ผนังหลัง → ประตู
[สูง][สั้น][สั้น]        [สูง][สั้น][สั้น]
  ↓    ↓    ↓               ↓    ↓    ↓
 เดิม ดี   ดี             [สูง][สั้น][สั้น]
```

**ทำไมต้องทำ:** ในการขนถ่ายจริง กองสั้นใกล้ประตูทำให้มองเห็นและหยิบสินค้าได้ง่ายกว่า  
นอกจากนี้ยังเป็นพื้นฐานให้โซน Condo ที่ประตูมีความสูงที่ถูกต้อง

---

### ขั้นที่ 6: การจัดวางโซนหน้าตู้ (Condo Placement)

**จุดประสงค์:** วางกล่องที่ยังเหลือ (ที่ Primary ไม่จุ) ในพื้นที่หน้าตู้ต่อจาก Primary Zone

**เงื่อนไขการเข้าโซน Condo:**
1. สินค้านั้นยังมีกล่องค้างเหลืออยู่หลัง Primary
2. จำนวนที่เหลือ ≥ `cols × targetLayers` (ต้องเต็ม 1 คอลัมน์พอดี)
3. มีพื้นที่ในแกน Y เพียงพอ

**วิธีคำนวณ:**

```
targetCondoZ = จำนวนชั้นสูงสุดที่ตู้รับได้ × ความสูงกล่อง
             = floor(InteriorH / H) × H

ตัวอย่าง: InteriorH=254, H=26.7 → floor(254/26.7) = 9 ชั้น → Z = 240.3 cm

cols = CondoCount (หรือ floor(InteriorW / W) ถ้าไม่ได้กำหนด)
targetLayers = floor(targetCondoZ / H)

วางกล่องเรียงเป็นคอลัมน์ (cols กว้าง × targetLayers สูง)
วนซ้ำจนหมดกล่องที่เหลือหรือหมดพื้นที่
```

**มองจากด้านข้าง:**

```
ผนัง ─────────────────────────── ประตู
      [Primary Zone]    [Condo]
      ████████████████  █████
      ████████████████  █████
      ████████████████  █████
      ████████████████  █████
      ────────────────  ─────
      ← ความยาว L−50  →← 50 →
```

> **หมายเหตุ:** Condo วางกล่องโดยไม่สลับ Pattern (วางตรงๆ เป็นแถว) เพราะเป็นบริเวณที่พนักงานขนถ่ายสินค้าออกก่อน

---

### ขั้นที่ 7: กระจายกล่องที่เหลือทับกองเดิม (Scatter Top Placement)

**จุดประสงค์:** ถ้ายังมีกล่องเหลืออยู่อีก (หลัง Primary + Condo) ให้นำไปวางทับบนยอดกองที่มีอยู่แล้ว  
**โดยเอากล่องของสินค้านั้นวางทับกองของสินค้านั้นเท่านั้น** (ห้ามข้ามสินค้า)

**วิธีการเลือกกอง:**

```
สร้างรายการ "กองที่รับได้" = กองที่ยังสูงไม่เต็มเพดาน

วนซ้ำจนกล่องหมดหรือกองเต็ม:
  1. เลือกกองที่ TopZ ต่ำสุด (สั้นที่สุด)
     ถ้าสูงเท่ากัน → เลือก StackIndex น้อยกว่า
  2. วางชั้นใหม่ที่ยอดกองนั้น (ใช้ Pattern ตาม A/B flip ของกองนั้น)
  3. อัปเดต TopZ ของกองนั้น
  4. ถ้ากองเต็มแล้ว → นำออกจากรายการ
```

**ตัวอย่างการเลือกกอง:**

```
รอบที่ 1:      รอบที่ 2:      รอบที่ 3:
████          ████          ████
████          ████  ████    ████  ████
████  ██      ████  ██      ████  ████
[A]  [B]      [A]  [B]      [A]  [B]

B สั้นกว่า    A สั้นกว่า    สูงเท่ากัน
→ วางบน B    → วางบน A    → วางบน A (SI เล็กกว่า)
```

**กฎสำคัญ:** Scatter **ไม่** จำกัดด้วย MaxLayers — วางได้จนเต็มความสูงตู้ (InteriorH)

## 5. รูปแบบการวางชั้น (Layer Pattern)

แต่ละชั้นถูกกำหนดด้วย **Pattern** ซึ่งประกอบด้วย Section หลายส่วนวางต่อกันในแนวกว้าง (X)

### โครงสร้าง Pattern

```
PatternA = [Section1, Section2, Section3, ...]
              │         │
              │         └─ SectionWidth 2
              └─ SectionWidth 1
              ← ความกว้างรวม = W1 + W2 + ... ≤ InteriorW
```

### โครงสร้าง Section

แต่ละ Section กำหนด SubRow ว่าวางกล่องกี่แถวกี่คอลัมน์ และหมุนหรือไม่:

```json
{
  "SubRows": [
    { "Rows": 1, "Cols": 3, "Rotated": false },
    { "Rows": 1, "Cols": 2, "Rotated": true  },
    { "Rows": 1, "Cols": 3, "Rotated": false }
  ]
}
```

**Rotated = true** หมายถึงหมุนกล่อง 90° (สลับแกน W↔L)

### ตัวอย่าง: Pattern Aloe 365 ML

```
PatternA: [Section(2 แถว × 6 คอลัมน์, ปกติ), Section(3 แถว × 3 คอลัมน์, หมุน)]
PatternB: [Section(3 แถว × 3 คอลัมน์, หมุน), Section(2 แถว × 6 คอลัมน์, ปกติ)]

ชั้น 0 (PatternA):
┌──────────────────────┬───────────┐
│  ○ ○ ○ ○ ○ ○        │ ○ ○ ○     │
│  ○ ○ ○ ○ ○ ○        │ ○ ○ ○     │
│                      │ ○ ○ ○     │
└──────────────────────┴───────────┘
     2×6 (ไม่หมุน)         3×3 (หมุน)

ชั้น 1 (PatternB):
┌───────────┬──────────────────────┐
│ ○ ○ ○     │  ○ ○ ○ ○ ○ ○        │
│ ○ ○ ○     │  ○ ○ ○ ○ ○ ○        │
│ ○ ○ ○     │                      │
└───────────┴──────────────────────┘
    3×3 (หมุน)      2×6 (ไม่หมุน)
→ สลับซ้าย-ขวา ทำให้แต่ละชั้นล็อกกันมั่นคงขึ้น
```

### Tier Replication (การทำซ้ำในแนวกว้าง)

ถ้าความกว้างตู้ (InteriorW) เกินกว่า 1 ชุด Pattern ระบบจะทำซ้ำ Pattern ในแนว X:

```
numTiers = floor(InteriorW / tierWidth)
tierWidth = ผลรวมความกว้างของทุก Section ใน Pattern

ตัวอย่าง: InteriorW = 239, tierWidth = 119 → numTiers = 2
→ วาง Pattern 2 ชุดเคียงกัน
```

## 6. ตัวอย่างจริง

### กรณี: Mogu 1000 ML × 540 + Mogu 320 ML × 850 ในตู้ 20 ft

**ข้อมูลเบื้องต้น:**

| รายการ | Mogu 1000 ML | Mogu 320 ML |
|--------|-------------|------------|
| ขนาด (W×L×H) | 26.2×34.8×26.7 | 25.8×38.5×15.7 |
| MaxLayers | 8 | 13 |
| ต้องการ | 540 กล่อง | 850 กล่อง |

**ผลลัพธ์ที่คาดหวัง (จาก Test):**

```
[ขั้น 0] EffectiveLayers:
  Mogu 1000: H=26.7, snap points = {26.7, 53.4, 80.1, ...}
  Mogu 320:  H=15.7, snap points = {15.7, 31.4, 47.1, ...}
  → หา Tₕ ที่ทำให้ใช้ความยาวตู้ไม่เกิน 545 cm (595−50)

[ขั้น 1] Primary Packing:
  Mogu 1000: วาง ~8 กอง × ~8 ชั้น = ~544 กล่อง (ครบ)
  Mogu 320:  วาง ~7 กอง × ~13 ชั้น = ~812 กล่อง

[ขั้น 2-4] Balancing:
  ปรับให้กองทุกกองของแต่ละสินค้ามีความสูงสม่ำเสมอ

[ขั้น 6] Condo:
  Mogu 320 ยังขาด ~38 กล่อง → วางในโซน Condo

[ขั้น 7] Scatter:
  ถ้ายังมีเหลือ → วางเสริมทับยอดกองเดิม
```

## 7. สรุปหลักการสำคัญ

### การรับประกันของ Algorithm

1. **ไม่มีกล่องล้นตู้** — ทุก BoxPlacement ถูกตรวจสอบให้อยู่ภายใน InteriorW × InteriorL × InteriorH
2. **ไม่วางเกินจำนวนที่ต้องการ** — Packed ≤ Requested ทุกสินค้า
3. **สินค้าไม่ข้ามกอง** — Scatter วางเฉพาะกล่องสินค้าเดิมทับกองของสินค้านั้นเท่านั้น
4. **ชั้นต้องต่อเนื่อง** — ไม่มีช่องว่างระหว่างชั้น ทุกกล่องมีฐานรองรับ

### ลำดับความสำคัญ

| ลำดับ | เป้าหมาย |
|------|---------|
| 1st | วางกล่องให้ได้มากที่สุด |
| 2nd | ใช้พื้นที่ตู้อย่างสม่ำเสมอ (ไม่กองสูงด้านเดียว) |
| 3rd | กองสั้นอยู่ใกล้ประตู (ขนง่าย) |
| 4th | ความสูงกองของสินค้าเดียวกันใกล้เคียงกัน (เสถียรภาพ) |

### ข้อจำกัดที่ทราบ

- สินค้าที่ **ไม่มี Pattern** (PatternA ว่าง) จะไม่ถูกวางเลย — ต้องกำหนด Pattern ก่อนใช้งาน
- **Condo** รองรับเฉพาะกล่องที่เหลือมากพอจนเต็ม 1 คอลัมน์ — ถ้าเหลือน้อยกว่านั้นจะใช้ Scatter แทน
- Algorithm ไม่รองรับการวางหลายสินค้าปนกันในกองเดียว — แต่ละกองมีสินค้าชนิดเดียว
